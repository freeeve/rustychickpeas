chore(release): v0.4.1

This release adds CSV file loading support and significantly improves Parquet
loading capabilities with better type handling, deduplication, and S3 integration.

## New Features

### CSV Loading Support
- Add `load_nodes_from_csv()` and `load_relationships_from_csv()` methods
- Support for both plain CSV and gzip-compressed CSV files (.csv.gz)
- Automatic type detection with optional explicit type hints via `CsvColumnType`
- Support for node deduplication based on unique properties
- Auto-generation of node IDs when ID column is not specified
- Full property and label support matching Parquet API

### Enhanced Parquet Loading
- Add `default_label` parameter to Parquet loading functions for applying
  default labels to all loaded nodes
- Improved handling of empty batches (now skipped automatically)
- Better support for various Arrow data types:
  - Int32 node IDs and relationship endpoints
  - Float32 property values
  - LargeUtf8 string columns
- Enhanced error handling for invalid column types
- Improved deduplication logic for nodes and relationships with null values

### Python API Improvements
- Add `default_label` parameter to `load_nodes_from_parquet()` in Python bindings
- Improved documentation for bulk loading methods

### S3 Integration
- Enhanced S3 integration tests with comprehensive property handling
- Support for loading nodes and relationships with properties from S3
- Tests for deduplication when loading from S3
- Better error handling for invalid S3 paths

## Improvements

- Extensive test coverage for edge cases in Parquet loading:
  - Empty batches
  - Various data types (Int32, Float32, LargeUtf8, etc.)
  - Deduplication with null values
  - Multi-property deduplication keys
  - Duplicate node IDs
  - Negative node IDs (error handling)
  - Null relationship types
- Code cleanup: remove unused type annotations
- Improved error messages for bulk loading operations

## Dependencies

- Add `csv = "1.3"` dependency for CSV parsing
- Add `flate2 = "1.0"` dependency for gzip decompression

## Testing

- Add comprehensive test suite for CSV loading
- Expand Parquet loading tests with 30+ new test cases
- Enhance S3 integration tests with property and deduplication scenarios
- Add tests for various data type combinations and edge cases

## Version

- Bump version from 0.4.0 to 0.4.1 in Cargo.toml and pyproject.toml
